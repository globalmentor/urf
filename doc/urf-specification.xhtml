Uniform Resource Framework (URF)

Version 2007-07-22

The Uniform Resource Framework (URF) provides a consistent way for representing resources and their relationships. URF and its interchange formats are useful for data storage, data interchange, data querying, and logical inferences. URF together with its interchange formats provides a more powerful yet simpler replacement for data-oriented XML, RDF, and JSON.

This document defines the Uniform Resource Framework (URF) as an abstract model, and also defines a text-based representation format, Text URF (TURF). This document does not yet provide a description of the binary representation format Serial URF (SURF), or the XML-based format XML URF (XMURF).

Conventions

When this document refers to "URF", it indicates the URF model abstract from any representation format. When this document refers to "TURF", it indicates the textual representation of URF. Because abstract concepts must be textualized to be communicated in a text-based specification, at times a description of URF will use TURF for clarification of the model.

Resources

URF

Everything that can be described by URF is referred to as a resource. Every resource is an instance of the class (<urf#Resource>).

TURF

A resource is indicated by parentheses. The following example refers to an anonymous resource:

()

Resource Identification

URF

A resource may be anonymous or may be identified by a URI.

TURF

A URI is indicated by angle brackets. The following example indicates the URI http://urf.name/urf#Resource:

<http://urf.name/urf#Resource>

A resource may be identified placing its URI within parentheses. The following example refers to a resource identified by the URI <http://urf.name/urf#Resource>.

(<http://urf.name/urf#Resource>)

Namespaces

URF

A set of resources (those part of a particular ontology, for example) may be grouped together for identification purposes by using a common namespace. A namespace is a URI that serves as a base for the formation of URIs within that namespace. A namespace URI may not have a fragment identifier. A URI within a namespace is formed by appending a pound sign followed by a local identifier to the namespace URI. For example, the URI <http://urf.name/urf#Resource> is within the namespace <http://urf.name/urf>.

TURF

A namespace may be given a prefix to use as a shortcut for indicating URIs within that namespace. A namespace prefix must consist only of name characters. The URI <http://urf.name/urf#Resource> can be represented using the prefix urf in place of the namespace URI <http://urf.name/urf> as in the following example:

Properties

URF

A resource may be related to another resource by the use of a property. Every property is itself a resource and is an instance of the class (<urf#Property>). A resource subject, a property predicate, and the related resource object together make an assertion about the universe being described.

An infinite number of assertions may be made about a single subject resource. An infinite number of these assertions may use the same predicate property and/or the same object resource. Within a particular context, the relative order of assertions about any particular resource subject is maintained.

TURF

Assertions about a particular subject resource are presented, in order, within angle brackets and separated by commas. Each predicate propertie and object resource of an assertion are separated by a colon character. In the following example, the resource (<http://example.org/example#resource>) has a single property (<http://example.org/example#property>), which has as its value another resource, (<http://example.org/example#value>):

(<http://example.org/example#resource>)
{
  (<http://example.org/example#property>):(<http://example.org/example#value>)
}

Were the prefix example defined to represent the namespace <http://example.org/example>, the above example could be represented in the following shorthand representation:

(<example#resource>)
{
  (<example#property>):(<example#value>)
}

Classes

URF

Some resources represent classes or sets of resources containing instances of other resources. As stated earlier, for example, all resources are implicitly instances of the class (<urf#Resource>), and all property resources are also implicitly instances of the class (<urf#Property>). Any resource may be explicitly asserted to be an instance of a class by identifying that class as a value of the (<urf#type>) property.

TURF

It can be asserted that a resource is an instance of a class by normal representation of the (<urf#type>) property. In the following example, the resource (<example#resource>) is asserted to be an instance of the class (<example#Class>):

(<example#resource>)
{
  (<urf#type>):(<example#Class>)
}
