Simple URF (SURF) Specification

Introduction

This section is non-normative.

SURF is a compact, text-based, human-readable persistence format. It is similar in purpose to JSON yet less verbose and more expressive, supporting for example a greater number of types. Moreover as one of the primary serializations (along with TURF) of the Uniform Resource Format (URF), it is compatible with rigorous semantic frameworks such as the Resource Description Framework (RDF).

Design Constraints

The following considerations were used to guide the creation of this specification:

  • SURF must support the Unicode character set for text values.
  • SURF should support the Unicode character set for identifiers and comments.
  • SURF must use only ASCII characters for delimiters.
  • All valid SURF documents must also be valid TURF documents, with the exception of introductory elements.
  • SURF must allow for distinctions among vocabularies of identifiers.
  • SURF must not require users to declare namespace URIs.
